== High-Level Requirements

[cols=".^1h,^.^2,.^2, options="header"]
|===
|  | Input | Description 

| Interface Name |  | The unique name of the interface
| Interface Description |  | Provide an overview of the interface's purpose and a short description from a business perspective
| Enterprise Data Object |  | This field is for the "simple" name of the data that the integration will be based on. The non-technical name of the data. For example 'Invoice', 'Order', etc.
|===

=== Sender

[cols=".^1h,^.^2,.^2, options="header"]
|===
|  | Input | Description 

| Sender system name |  | The name of the system
| Sender protocol | | SOAP, HTTP (for RESTful services), oData, SFTP, Kafka, SAP RPC, etc.
| Message format | | JSON, XML, CSV, etc.
| Technical object / API name | | The name of the technical objects in the sender system
| Rationale | | Describe reasons for selecting both message protocol and format
|===

=== Integration characteristics

[cols=".^1h,^.^2,.^2, options="header"]
|===
|  | Input | Description 

| Interaction type |  | Sync or Async
| Pull or Push |  | Indicate whether the integration is initiated by the sender (Push) or by the receiver (Pull)
| Frequency |  | How often the integration is expected to run. For example, event-based, scheduler-based (specify daily, weekly, monthly, etc.)
| Volumetry. Average message size | | The average size of the message in KB, MB, etc.
| Volumetry. Average message frequency | | The average number of messages per day, week, month, etc.
| Volumetry. Peak message size | | The peak size of the message in KB, MB, etc.
| Volumetry. Peak message frequency | | The peak number of messages per day, week, month, etc.
| Security requirements | | Describe the security requirements for the integration
| Message transformation | | Describe the transformation that needs to be done on the message
| Existing consumers | | List of the existing consumers of the interface
| Additional information | | Any additional information that is relevant to the integration
|===
